# Exploration Fast Track

## 规则
`explorations/` 目录是研究沙箱，规则宽松:
- 质量阈值降低: 60/100 即可 (正式代码需 80)
- 无需事先计划 (可直接动手)
- 可随时终止 (不需要完成整个循环)
- 代码风格要求宽松 (可用 notebook 风格)

## 要求
- 每个探索有自己的子目录: `explorations/<name>/`
- 每个探索有 README.md: 记录目的、方法、结论
- 数据安全规则仍然适用 (look-ahead bias 仍然零容忍)

## 毕业条件
当探索产出有价值的结果:
1. 质量评分 >= 80
2. 代码整理到正式目录 (delta_alpha/, delta_learn/ 等)
3. 在 README 中标注 "graduated to [path]"
4. 保留探索目录作为记录, 不删除
