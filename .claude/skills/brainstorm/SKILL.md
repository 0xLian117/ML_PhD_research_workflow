# /brainstorm

## 目标

用户提出一个粗略想法后, 对外部资源进行**广泛深入调研**, 然后:
1. 评估想法的可行性和新颖性
2. 指出潜在问题和盲点
3. 提出改进建议和替代方案
4. 给出具体可行的下一步

## 步骤

### 1. 理解想法

从用户描述中提取:
- **核心想法**: 用户想做什么
- **动机**: 为什么想做这个
- **预期目标**: 希望达到什么效果
- **约束条件**: 技术栈、数据、算力等限制

如信息不足, 先向用户追问关键细节后再继续。

### 2. 大规模并行调研

使用 Agent 工具并行启动多个 literature-assistant agent 进行搜索, 覆盖三类来源:

#### 2a. 学术论文
搜索策略 (至少 3 轮):
- `"[核心概念] survey/review [2024-2026]"` — 最新综述
- `"[具体方法] [应用场景]"` — 直接相关工作
- `"[替代方法] [同一问题]"` — 竞争方案
- 优先搜索 arXiv, NeurIPS, ICML, ICLR, KDD, ACL 等顶会

对每篇重要论文记录:
- 标题、作者、年份、venue
- 核心方法 (一句话)
- 与用户想法的关系: 支持 / 竞争 / 互补
- 关键结果和数字

#### 2b. 代码实现
搜索策略:
- `"[方法名] site:github.com"` — 开源实现
- `"[方法名] implementation/code"` — 代码资源
- `"[框架] [方法名] example"` — 具体框架实现

对每个重要 repo 记录:
- 仓库名和 URL
- Star 数 (如可见)
- 实现语言/框架
- 成熟度评估: 生产级 / 研究代码 / demo

#### 2c. 技术博客和讨论
搜索策略:
- `"[方法名] tutorial/blog/guide"` — 教程和经验
- `"[方法名] vs [替代方法]"` — 方法对比
- `"[方法名] pitfalls/limitations"` — 已知问题和坑

对每篇重要博客记录:
- 标题和 URL
- 核心观点
- 实践经验和教训

### 3. 综合分析

基于调研结果, 进行结构化分析:

```markdown
## 调研发现摘要

### 支持证据
- [哪些研究/实践支持这个方向]

### 已有类似工作
- [谁做过类似的事, 效果如何, 与用户想法的差异]

### 潜在问题和风险
- [文献中报告的失败案例或局限性]
- [理论上可能的瓶颈]
- [工程实现上的已知难点]

### 值得参考的方法/技巧
- [调研中发现的有用技巧, 用户可能没想到的]
```

### 4. 建议和修正

基于分析结果, 给出**具体且可操作**的建议:

```markdown
## 对原始想法的建议

### 建议保留的部分
- [想法中哪些方面是合理且有潜力的]

### 建议修改的部分
- [问题 1]: [具体修改建议 + 理由 + 参考文献/代码]
- [问题 2]: ...

### 新增建议
- [调研中发现的、用户未考虑到的改进点]
```

### 5. 可行方案

提出 2-3 个具体方案, 从保守到激进排列:

```markdown
## 可行方案

### 方案 A: [名称] (保守/低风险)
- **思路**: ...
- **参考**: [论文/代码/博客]
- **优势**: ...
- **劣势**: ...
- **预估工作量**: ...

### 方案 B: [名称] (均衡)
- **思路**: ...
- **参考**: [论文/代码/博客]
- **优势**: ...
- **劣势**: ...
- **预估工作量**: ...

### 方案 C: [名称] (激进/高风险高回报)
- **思路**: ...
- **参考**: [论文/代码/博客]
- **优势**: ...
- **劣势**: ...
- **预估工作量**: ...

### 推荐
[基于用户约束条件, 推荐哪个方案, 为什么]
```

### 6. 资源汇总

```markdown
## 参考资源

### 必读论文 (Top 3-5)
1. [Title] (Author, Year) — [一句话说明为什么必读]

### 推荐代码库
1. [repo_name](url) — [一句话说明]

### 推荐阅读 (博客/教程)
1. [Title](url) — [一句话说明]
```

### 7. 保存

保存完整调研报告到:
`explorations/brainstorm_YYYY-MM-DD_[topic_slug].md`

### 8. 互动讨论

向用户展示:
- 调研发现的**核心洞察** (3-5 条)
- 推荐方案及理由
- 最关键的参考资源 (前 3 个)
- 询问: 用户倾向哪个方案, 或是否需要在某个方向进一步深挖

**注意**: WebSearch 结果需用户验证准确性。论文和代码信息可能不完整或过时, 建议用户对关键参考进行原文确认。
